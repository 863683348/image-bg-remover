# Image Background Remover — MVP 需求文档

> **版本：** v1.0
> **日期：** 2026-07-10
> **状态：** 草案

## 1. 产品概述

### 1.1 愿景
让用户无需安装任何软件、无需专业技能，在浏览器中几秒内完成图片去背景操作。

### 1.2 产品定位
一款轻量、即用即走的在线去背景工具。不做复杂的编辑器，聚焦"上传 → 去背景 → 下载"这一核心链路，把每一步做到极致流畅。

### 1.3 目标用户
| 用户画像 | 典型场景 |
|---|---|
| 电商卖家 / 微商 | 给商品图去背景做白底图 |
| 社交媒体运营 | 快速抠人像/产品做封面 |
| 设计师 / 创作者 | 前后期素材预处理 |
| 普通用户 | 证件照换背景、简单抠图需求 |

## 2. MVP 范围

### 2.1 核心功能（In Scope）
| # | 功能 | 优先级 | 说明 |
|---|---|---|---|
| F1 | 图片上传 | P0 | 支持点击选择 + 拖拽上传，格式 JPG / PNG / WebP |
| F2 | 去背景处理 | P0 | 通过 Remove.bg API 完成，图片最大 25MB |
| F3 | 结果预览 | P0 | 棋盘格显示透明度，原图与结果并排对比 |
| F4 | 背景色预览 | P1 | 透明 / 纯色（预设 6 色 + 自定义取色器） |
| F5 | 下载结果 | P0 | 下载透明 PNG 或带背景色的 PNG |
| F6 | 错误处理 | P0 | 上传校验、API 失败提示、超时处理 |
| F7 | 换一张图 | P1 | 从结果页一键回到选图流程 |
| F8 | 响应式布局 | P1 | 桌面优先，适配平板和手机 |
| F9 | 暗色模式 | P2 | 跟随系统主题自动切换 |

### 2.2 非功能需求
| # | 需求 | 指标 |
|---|---|---|
| N1 | 页面加载 | 首次加载（不含 API）< 2s |
| N2 | API 响应 | 去背景耗时取决于 Remove.bg，前端需展示加载状态 |
| N3 | 图片上传 | 拖拽即上传，无额外确认步骤 |
| N4 | 浏览器兼容 | 支持 Chrome / Firefox / Safari / Edge 最新两个大版本 |
| N5 | 可用性 | 服务正常运行时间 >= 99.9%（依赖 Cloudflare Pages + Remove.bg） |
| N6 | 隐私 | 图片经内存处理，不落盘；删除即为销毁 |

### 2.3 明确不包含（Out of Scope for MVP）
- 用户注册/登录系统
- 图片历史记录管理
- 批量处理
- AI 本地模型（当前用云 API）
- 图片裁剪 / 旋转 / 滤镜等编辑功能
- 图片存储/CDN 缓存

## 3. 用户流程

```
  进入页面 -> 上传图片 -> 预览原图 -> 点击去背景
                                      |
                                      v
                              等待处理（加载态）
                                      |
                                      v
                              展示结果 -> 切换背景色
                                      |
                                      v
                                   下载 PNG
                                      |
                                      v
                               点击"新图片"-> 回到上传
```

### 页面状态流转

| 状态 | 触发条件 | 界面表现 |
|---|---|---|
| 初始态 | 首次加载 | 拖拽上传区，无图片 |
| 预览态 | 用户选图完成 | 原图显示，Remove Background 按钮可用 |
| 处理中 | 点击去背景 | 全屏遮罩 + spinner |
| 结果态 | API 返回成功 | 原图/结果并排，背景色选择器 + 下载按钮 |
| 错误态 | API 失败 / 上传校验失败 | 底部 Toast 错误提示，可关闭 |

## 4. 技术架构

### 4.1 架构决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 部署平台 | Cloudflare Pages + Functions | 零服务器运维，全球 CDN，天然 HTTPS |
| 去背景 API | Remove.bg | 成熟、准确率高，有免费额度 |
| 前端技术 | Vanilla HTML/CSS/JS | 无框架依赖，构建即产物，极致轻量 |
| 数据存储 | 无（纯内存） | 隐私优先，零存储成本 |
| 图标库 | Lucide | 轻量、SVG 按需加载、图标齐全 |
| 本地开发 | Node.js 纯静态服务器 | 替代 workerd（Windows 兼容性问题） |

### 4.2 数据流

```
[用户] drag/drop/click
    |
    v
[浏览器] FileReader -> 本地预览
    |
    v 点击 Remove Background
[浏览器] FormData(image) ---POST--> [Cloudflare Pages Function]
                                       |
                                       | X-Api-Key (环境变量)
                                       v
                                   [Remove.bg API]
                                       |
                                       | 返回 PNG
                                       v
[浏览器] <---- 透明 PNG image/png ---- [Cloudflare Pages Function]
    |
    v
[浏览器] 显示结果 -> 用户选择是否替换背景色 -> 下载
```

### 4.3 API 接口定义

#### POST /api/remove-bg

| 字段 | 类型 | 说明 |
|---|---|---|
| Request Body | multipart/form-data | 包含字段 image（文件） |
| Response (200) | image/png | 去背景后的透明 PNG |
| Response (4xx/5xx) | application/json | { "error": "...", "detail": "..." } |

错误码对照：
- 400 - 无图片 / 非图片文件 / 超过 25MB
- 402 - Remove.bg API 额度不足
- 500 - API Key 未配置 / 服务器内部错误

## 5. 用户界面设计

### 5.1 设计原则
- 工具感优先：不做营销页面，用户进入即可操作
- 全链路无障碍：每个状态都有清晰反馈（加载、成功、失败）
- 极简交互：能一步完成绝不两步

### 5.2 页面布局示意

```
+------------------------------------------------------------------+
| [*] Background Remover                  <- 顶部导航栏，固定       |
+------------------------------------------------------------------+
|                                                                  |
|     +----------------------------------------------------+       |
|     |                                                    |       |
|     |    [image-up]  Drop image here                     |       | <- 拖拽上传区
|     |    or click to browse                              |       |    虚线边框 + hover 高亮
|     |                                                    |       |
|     +----------------------------------------------------+       |
|                                                                  |
+------------------------------------------------------------------+
|                                                                  |
|  +--------------+    +--------------+                             |
|  | Original     |    | Result       |                             | <- 并排对比
|  | [img]        |    | [checker]    |                             |    （响应式变纵向）
|  +--------------+    +--------------+                             |
|                                                                  |

|                                                                  |

```

### 5.3 交互细节
- 拖拽经过上传区时，边框变为主题色 + 背景浅色高亮
- 处理状态的 spinner 使用主题色的旋转圆环
- 下载有背景色的图片时，前端 Canvas 合成后再触发下载
- 错误 Toast 5 秒后自动消失，也可手动关闭

## 6. 部署方案

### 6.1 环境要求

| 资源 | 说明 |
|---|---|
| Remove.bg API Key | 免费注册，每月 50 次调用额度 |
| Cloudflare 账号 | 免费套餐支持 Pages 和 Functions |
| Node.js（可选） | 仅在本地开发和首次部署时需要 |

### 6.2 部署步骤

1. 注册 remove.bg -> 获取 API Key
2. 注册 Cloudflare -> 连接 Git 仓库
3. 在 Cloudflare Dashboard 设置环境变量: REMOVE_BG_API_KEY = "your-key"
4. 部署: npx wrangler pages deploy . --project-name image-bg-remover
5. 可选: 绑定自定义域名

### 6.3 CI/CD（后续迭代）
- GitHub Actions 监听 main 分支，自动部署到 Cloudflare Pages
- 预览部署用于 PR 分支测试

## 8. 风险评估

| 风险 | 影响等级 | 缓解措施 |
|---|---|---|
| Remove.bg API 服务不可用 | 高 | 预留切换至其他 API 的抽象层，降级提示 |
| API 免费额度耗尽 | 中 | 调用前检查余额，超出后提示用户自行配置 Key |
| 大图（>10MB）处理超时 | 中 | 前端压缩后上传，展示进度条 |
| Worker 运行时限制（10ms CPU） | 低 | Functions 只做透传，不涉及重计算 |
| 用户上传敏感图片 | 低 | 隐私声明说明图片不存储，走内存处理 |
